📘 Guide Comut

Ce tutoriel vous guide pas-à-pas pour déployer les 3 projets Comut : le backend sur Render, le site sur GitHub Pages, et l'APK Android via Expo EAS Build.

✅ Testé et fonctionnel ⏱ ~30 min de déploiement 📱 Android uniquement 🆓 Gratuit (plans free)
★

Les 3 projets

⚙️
comut-backend
API REST Node.js. Gère l'auth, les groupes, les uploads et les médias.
Node.js Express MongoDB Render
📱
comut-app
Application mobile React Native. Build en APK via Expo EAS Build cloud.
React Native Expo EAS Build
🌐
comut-web
Site vitrine hébergé sur GitHub Pages. Présente l'application.
HTML/CSS GitHub Pages
1

Créer le repo comut-backend sur GitHub

Vous allez créer un nouveau repo GitHub public et y uploader le dossier comut-backend depuis le ZIP téléchargé.

1
Créer le repo sur GitHub
Allez sur github.com → New repository. Nom : comut-backend. Visibilité : Public. Ne cochez rien d'autre. Cliquez Create repository.
2
Extraire le ZIP et initialiser Git
Extrayez le ZIP comut-backend.zip sur votre ordinateur. Ouvrez un terminal dans le dossier comut-backend et exécutez :
bash
cd comut-backend
git init
git add -A
git commit -m "🚀 Initial commit: Comut Backend v1.0"
git branch -M main
git remote add origin https://github.com/alexbg577/comut-backend.git
git push -u origin main
ℹ️
Authentification Git
Si Git vous demande un mot de passe, utilisez votre Personal Access Token GitHub (pas votre mot de passe habituel). Settings → Developer settings → Personal access tokens → Fine-grained tokens.

Déployer sur Render

Render va détecter automatiquement votre repo et le déployer. Le fichier render.yaml est déjà configuré.

1
Créer un compte Render
Allez sur render.com → Sign up with GitHub → Autorisez l'accès.
2
Nouveau Web Service
Tableau de bord Render → New + → Web Service → Connect a repository → sélectionnez alexbg577/comut-backend.
3
Configurer le service
Remplissez ces champs :
config render
Name          : comut-backend
Environment   : Node
Build Command : npm install
Start Command : node src/index.js
Plan          : Free
4
Ajouter les variables d'environnement
Avant de cliquer "Create", faites défiler jusqu'à Environment Variables et ajoutez toutes les variables du tableau ci-dessus une par une.
✅
Déploiement automatique
Une fois créé, chaque push sur main redéploiera automatiquement le backend. Le premier déploiement prend ~3 minutes.
5
Vérifier que l'API fonctionne
Ouvrez dans votre navigateur :
url
https://comut-backend.onrender.com/
Vous devez voir : {"status":"Comut API ✅","version":"1.0.0"}
😴
Plan Free — Cold Start
Sur le plan gratuit Render, le serveur s'endort après 15 minutes d'inactivité. Le premier appel après inactivité prendra ~30 secondes. Pour éviter ça, upgradez vers le plan Starter ($7/mois) ou utilisez un service comme UptimeRobot pour pinger l'API toutes les 10 minutes.

Installer les outils

Vous avez besoin de Node.js et de l'outil EAS CLI d'Expo pour builder l'APK en cloud.

1
Installer Node.js 20+
Téléchargez Node.js LTS depuis nodejs.org. Vérifiez l'installation :
bash
node --version   # doit afficher v20.x.x ou supérieur
npm --version    # doit afficher 10.x.x ou supérieur
2
Installer Expo CLI et EAS CLI
bash
npm install -g expo-cli eas-cli
3
Créer un compte Expo
Allez sur expo.dev → Sign up (gratuit). Puis connectez-vous en ligne de commande :
bash
eas login   # Entrez votre email et mot de passe Expo

Configurer l'app

1
Extraire et installer les dépendances
bash
cd comut-app
npm install
2
Lier à votre compte Expo
Cette commande crée le projet sur expo.dev et met à jour app.json :
bash
eas init   # Répondez "comut" quand il demande le nom du projet
3
Vérifier l'URL du backend
Ouvrez services/api.js et vérifiez que l'URL pointe bien vers votre backend Render :
javascript
const BASE_URL = 'https://comut-backend.onrender.com/api';
💡
URL différente ?
Si Render vous a donné une URL différente (ex: comut-backend-xxxx.onrender.com), mettez-la à jour ici ET dans services/upload.js.

Build APK avec EAS

EAS Build compile votre app dans le cloud — pas besoin d'Android Studio ni de Mac.

1
Lancer le build APK
bash
eas build --platform android --profile preview
Il vous demandera si vous voulez créer un keystore Android — répondez Y (Expo le gère automatiquement).
2
Attendre le build (5-15 min)
Le build se fait dans le cloud. Vous pouvez suivre la progression sur expo.dev → votre projet → Builds. Une fois terminé, un lien de téléchargement APK apparaît.
🎉
APK prêt !
EAS vous envoie un email quand le build est terminé. Vous pouvez aussi retrouver le lien sur expo.dev ou en relançant eas build:list.
3
Télécharger l'APK
bash
# Lister les builds récents
eas build:list

# Ou télécharger directement depuis expo.dev
# expo.dev → Votre projet → Builds → Download
9

Installer l'APK sur Android

1
Activer les sources inconnues
Sur Android : Paramètres → Sécurité → Installer des applications inconnues → activez pour votre navigateur ou gestionnaire de fichiers.
2
Transférer l'APK
Envoyez l'APK par email, WhatsApp ou câble USB. Ou scannez le QR code depuis expo.dev directement depuis votre téléphone.
3
Installer et ouvrir
Appuyez sur le fichier APK → Installer → Ouvrir. L'app Comut apparaît sur votre écran d'accueil.

FAQ — Problèmes fréquents

Q
Le backend ne démarre pas sur Render
Vérifiez les logs Render. Cause la plus fréquente : variable d'environnement manquante, notamment MONGODB_URI. Assurez-vous d'avoir ajouté toutes les variables dans Settings → Environment.
Q
L'email de vérification n'arrive pas
Sur le plan Resend gratuit sans domaine vérifié, les emails ne peuvent être envoyés qu'à votre propre adresse (pelassyalexandre@gmail.com). Pour les autres utilisateurs, ajoutez un domaine vérifié sur resend.com et mettez à jour le champ from dans src/utils/email.js.
Q
EAS build échoue
Vérifiez que vous avez bien lancé eas init avant le build. Si l'erreur concerne un module natif, nettoyez le cache :
bash
eas build --platform android --profile preview --clear-cache
Q
Upload de fichiers échoue (> 100MB)
Cloudinary gratuit limite les uploads à 100 MB par fichier (pas 3 GB). Pour les gros fichiers, passez au plan Cloudinary payant ou réduisez la limite dans src/routes/content.js.
Q
L'app affiche "Erreur réseau"
Le backend Render est probablement en "cold start" (endormi). Attendez 30 secondes et réessayez. Vous pouvez utiliser UptimeRobot (gratuit) pour pinger l'API toutes les 10 minutes et éviter l'endormissement.
Q
Mettre à jour l'app après modifications
Pour le backend : faites un git push et Render redéploie automatiquement. Pour l'app mobile : relancez un build EAS et redistribuez l'APK.
bash
# Mise à jour backend
git add -A && git commit -m "update" && git push

# Nouveau build APK
eas build --platform android --profile preview
